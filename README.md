# first-time github
##learning-log
自分で認証局（CA）を立てるとは？
「自分で認証局（CA）を立てる」とは、例えるなら「自分専用の役所を作って、身分証明書（デジタル証明書）を自前で発行する」ようなものです。
通常、ネット上の証明書は「デジサート」や「セコム」といった、世界的に信頼された「公的な役所（公的認証局）」が発行しますが、特定の目的のために、自分のPCやサーバーを「役所」として機能させることができます。
1. なぜ自分で立てるのか？（主な理由）
社内・家庭内ネットワークの安全強化
外部に公開しない社内Wi-Fiやサーバーで、許可した端末だけを繋ぎたい場合に利用します。
コスト削減
公的な証明書は管理や発行に費用がかかることがありますが、身内だけで使うなら自前（無料）で十分です。
開発・テスト用
エンジニアがアプリを作る際、本物の証明書を購入する前に「通信が暗号化されるか」を試すために作成します。
2. どうやって作るのか？（技術的な手段）
専門的なソフトを使って、自分のPCを「証明書発行マシン」に変えます。
OpenSSL: 最も有名な無料ツール。コマンドを入力して作成します。
Windows Server: 企業向け機能として「Active Directory 証明書サービス」があります。
Google Cloud / AWS: クラウド上で「プライベート認証局」を簡単に構築できるサービスを提供しています。
3. 注意点：自作の証明書は「怪しまれる」
自作の証明書（通称：オレオレ証明書）をスマホにインストールしても、スマホ側は「この役所（あなたのPC）は知らない場所だ！」と警告を出します。
そのため、スマホ側の設定で「この自作の役所（認証局）は信頼できる場所である」とあらかじめ登録（インポート）しておく作業が必要になります。
